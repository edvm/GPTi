GPTI | ChatGPT on your command line

Get started with GPTI

GPTi is a tool built in Rust that lets you interact with the OpenAI GPT-3 model from the comfort of your command line. Additionally, you can pre-save your own prompts by defining a gpti.toml config file in order to avoid typing the same prompts over and over again in the web browser when using chatgpt. Also, its free (OpenAI doesn't charge for using GPT-3).

Why did I create this project?

  • I'm a heavy terminal/cli user, so it's more conforable to ask ChatGPT from my command line (instead of having to open it on the web browser each time)
  • There are some prompts that I use very often, so instead of typing them over and over again, I can just save them in a config file and use them with a single command
  • I wanted to learn/practice Rust

Quick Start

Install via curl (only mac arm or linux):

            
  $ bash -c "$(curl -fsSL https://github.com/edvm/GPTi/raw/main/docs/releases/install.sh)"

Or install via Cargo:

            
  $ git clone https://github.com/edvm/GPTi.git
  $ cd GPTi && cargo build --release

Source code is available on GitHub.

Usage

Run gpti -h to see the available commands:

            
GPTi

Usage: gpti [OPTIONS]

Options:
  -c, --config   Config file (optional)
  -p, --prompt   Prompt name to use (from config file)
  -t, --text       Prompt text to use (inlined, text will be sent to OpenAI as is)
  -h, --help             Print help
  -V, --version          Print version info
First time

For the first time, run gpti -p init to create a gpti.toml config file. It'll be created with a sample prompt. Later, you must add your OpenAI API api key to the config file. In order to get an OpenAI API Key (its free), check this link and follow the instructions on OpenAI website.

gpti -p <prompt_name>

To use a prompt from the config file, run gpti -p <prompt_name> This is useful if you want to use a prompt that you use often. For example, if you want to use the prompt grammar that you defined in your config file, you can run gpti -p grammar. When running this command, you'll be prompted to enter the text that you want to send to OpenAI. Sent text will be: text from config file + text you enter. Once you're done, type EOF to finish or press ENTER more than three times. Output will be printed to stdout and will be copied to your clipboard.

gpti -t "2 + 2?"

To use a prompt inlined, run gpti -t <text you'd write on OpenAI website (Message ChatGPT...)>. This is useful if you want to use a prompt that you don't want to save in the config file.

Configuration

GPTi uses a gpti.toml file to store your prompts. You can create this file by running gpti -p init (for the first time). The format of the file is as follows: 

            
[openai]
api_key = ""

[[prompt]]
name = "grammar"
text = '''Fix any grammar issues in the following text wrapped by triple backticks.'''
description = "Fix grammar issues in the following text. Type EOF to finish."

You can add as many prompts as you want. The name field is the name of the prompt that you will use when running gpti -p. The text field is the prompt text that will be sent to OpenAI. The description field is just a description of the prompt that will be shown when running gpti -p.
